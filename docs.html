Introdução

Essa API surgiu da nossa necessidade de fazer aplicativos para o Eventick. Iniciamente a unica escrita permitida é o checkin de participantes. Tentamos seguir ao máximo as orientaçoes de Design RESTful.

A API está disponivel somente via HTTPS. A URL base é https://www.eventick.com.br/api/v1. Todos os dados devem ser recebidos e enviados em JSON. Os campos de data e tempo estarão no formato ISO 8601: YYYY-MM-DDTHH:MM:SS±hh:mm

Erros

Utilizamos os status code do protocolo http, que estarão disponível no header de cada response.

200 OK
Tudo funcionou como esperado
400 Bad Request
Geralmente um problema com os parametros.
401 Unauthorized
Acesso com token de acesso invalido.
402 Request Failed
Parameters were valid but request failed.
404 Not Found
O recurso acessado não existe.
50X Server errors
Nossa falta, algum erro do nosso servidor.

Autenticação

Ao executar uma autenticação você receberá um token para as demais requisiçoes. A autenticação é via HTTP Basic Auth.

Resource URL

GET https://www.eventick.com.br/api/v1/tokens.json

Parâmetros

email
Obrigatório.

Você deve informar o email que você utiliza para acessar sua conta no Eventick.

password
Obrigatório.

Senha utlizada para acessa sua conta no eventick

Exemplo de Requisição

$ curl -i https://www.eventick.com.br/api/v1/tokens.json \
    -u "jesus@eventick.com.br:12345678"

Exemplo de Resposta

{
    "token":"dpoi2154wijdsk4fo65ow4o2pkd"
}

Listagem de Participantes

Resource URL

GET https://www.eventick.com.br/api/v1/attendees.json

Parâmetros

event_id
Obrigatório.

O Identificado do evento que você deseja lista os participantes

auth_token
Obrigatório.

Token necessario para identificação do usuário

$ curl -i https://www.eventick.com.br/api/v1/attendees.json \

Checkin de Participante

You must include a "Content-Type: application/json" header to indicate this for each request that includes any data.

Resource URL

PUT https://www.eventick.com.br/api/v1/attendees/check_all.json

Parâmetros

attendees

Os attendees para serem checados devem ser enviados nos formato JSON. Ex: { "id": 10, "checked_at": "2012-10-17T16:54:35-03:00"}

auth_token
Obrigatório.

Token necessário para identificação do usuário

Exemplo de Requisição

$ curl -i https://www.eventick.com.br/api/v1/attendees/check_all.json \
    -H "Content-Type: application/json" \ 
    -X PUT \
    -u "f9d93dk211394:" \
    -d '{"attendees":[{"id":10,"checked_at":"2012-10-17T16:54:35-03:00"}]}'

Exemplo de Resposta

{
    "attendees": [
        {
            "id": 10,
            "checked_at": "2012-10-17T16:54:35-03:00"
        },
        {
            "id": 11,
            "checked_at": "2012-10-17T16:54:35-03:00"
        }
    ]
}
